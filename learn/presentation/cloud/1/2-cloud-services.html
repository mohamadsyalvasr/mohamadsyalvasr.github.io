Model Layanan Cloud

Membedah IaaS, PaaS, dan SaaS menggunakan analogi "Pizza as a Service".

INFRASTRUCTURE AS A SERVICE

IaaS: Membuat Pizza dari Awal

Bayangkan Kamu memiliki dapur dengan kompor dan oven (infrastruktur), tetapi semua bahan—tepung, tomat, keju—Kamu siapkan sendiri. Kamu membuat adonan, saus, dan menyusun pizza dari nol.

Dalam dunia cloud, IaaS memberikan Kamu blok bangunan dasar: server virtual, penyimpanan, dan jaringan. Kamu bertanggung jawab penuh untuk menginstal sistem operasi, perangkat lunak, dan aplikasi.

Contoh: Amazon Web Services (EC2), Microsoft Azure (VM), Google Compute Engine.

PLATFORM AS A SERVICE

PaaS: Pizza Kit Setengah Jadi

Sekarang, bayangkan Kamu membeli "pizza kit". Adonan dan saus sudah disiapkan oleh penjual. Tugas Kamu hanya menambahkan topping favorit Kamu dan memanggangnya di oven yang juga sudah disediakan.

PaaS menyediakan platform (sistem operasi, database, server web) yang siap pakai. Kamu hanya perlu fokus pada pengembangan dan pengelolaan aplikasi Kamu, tanpa pusing memikirkan infrastruktur dasarnya.

Contoh: Heroku, Google App Engine, AWS Elastic Beanstalk.

SOFTWARE AS A SERVICE

SaaS: Pesan Antar Pizza Jadi

Ini adalah opsi termudah. Kamu tinggal menelepon atau membuka aplikasi, memesan pizza, dan pizza tersebut diantar ke rumah Kamu dalam keadaan panas dan siap santap. Kamu tidak terlibat dalam proses pembuatannya sama sekali.

SaaS adalah perangkat lunak yang sepenuhnya dikelola oleh penyedia dan siap Kamu gunakan melalui internet. Kamu hanya perlu login dan menggunakannya, seperti Gmail, Dropbox, atau Salesforce.

Contoh: Google Workspace, Microsoft 365, Slack, Dropbox.

Penyedia Mengelola:

  • Runtime
  • Middleware

  • Virtualisasi
  • Server
  • Penyimpanan
  • Jaringan

Kamu Mengelola:

  • (Tidak ada)

Penyedia Mengelola:

  • Semuanya

Apa Selanjutnya?

Kamu kini dapat membedakan tiga model layanan utama dalam Cloud Computing. Setiap model menawarkan tingkat kontrol dan kemudahan yang berbeda, sesuai dengan kebutuhan spesifik Kamu.

Pada episode berikutnya, kita akan membahas Model Deployment: Public, Private, dan Hybrid Cloud.
