Episode 3

Model Deployment Cloud

Memahami perbedaan antara Public, Private, dan Hybrid Cloud.

PUBLIC CLOUD

Tinggal di Apartemen

Infrastruktur cloud dimiliki dan dioperasikan oleh penyedia pihak ketiga (seperti AWS, Google, Microsoft) dan sumber dayanya (seperti server dan penyimpanan) digunakan bersama oleh banyak organisasi melalui internet.

Ini seperti tinggal di gedung apartemen. Anda berbagi fasilitas (listrik, air, keamanan) dengan penghuni lain. Biayanya lebih rendah dan Anda tidak perlu pusing soal pemeliharaan gedung.

Kelebihan: Biaya rendah, skalabilitas tinggi, tidak perlu pemeliharaan.

PRIVATE CLOUD

Memiliki Rumah Sendiri

Sumber daya komputasi awan digunakan secara eksklusif oleh satu bisnis atau organisasi. Cloud ini bisa berlokasi di pusat data internal perusahaan atau di-hosting oleh penyedia layanan pihak ketiga.

Ini seperti memiliki rumah pribadi. Anda memiliki kontrol penuh atas keamanan, desain, dan semua isinya. Namun, semua biaya pembangunan dan pemeliharaan menjadi tanggung jawab Anda.

Kelebihan: Kontrol penuh, keamanan dan privasi yang lebih tinggi.

HYBRID CLOUD

Rumah dengan Paviliun

Model ini menggabungkan cloud publik dan cloud privat, memungkinkan data dan aplikasi untuk berpindah di antara keduanya. Ini memberikan fleksibilitas dan lebih banyak opsi deployment.

Bayangkan Anda memiliki rumah pribadi (Private Cloud) tetapi juga menyewakan paviliun (Public Cloud) saat ada tamu tambahan. Data sensitif tetap di rumah utama, sementara aplikasi yang butuh skalabilitas tinggi bisa ditaruh di paviliun.

Kelebihan: Fleksibilitas, skalabilitas, keamanan yang seimbang.

Apa Selanjutnya?

Kini Anda telah memahami tiga model deployment utama dalam komputasi awan. Pemilihan model yang tepat bergantung pada kebutuhan spesifik terkait biaya, kontrol, dan skalabilitas.

Pada episode berikutnya, kita akan membahas Manfaat dan Risiko utama dalam mengadopsi teknologi Cloud.
