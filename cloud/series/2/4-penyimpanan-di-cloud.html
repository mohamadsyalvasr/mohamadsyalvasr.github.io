Seri 2: Cloud dalam Praktik

Penyimpanan di Cloud

Memilih 'lemari' yang tepat untuk data Anda: Block, File, dan Object Storage.

Tiga Jenis Utama Penyimpanan

1. Block Storage

**Analogi: Hard Disk (Drive C:).** Ini adalah penyimpanan berkinerja tinggi yang melekat langsung ke satu VM. Digunakan untuk menginstal sistem operasi dan database yang butuh akses baca/tulis super cepat. Tidak bisa diakses oleh banyak VM sekaligus.

2. File Storage

**Analogi: Folder Sharing Kantor (NAS).** Ini adalah sistem file terpusat yang bisa diakses oleh banyak VM secara bersamaan. Cocok untuk menyimpan dokumen atau aset yang perlu digunakan bersama oleh beberapa server aplikasi.

3. Object Storage

**Analogi: Gudang Raksasa Tak Terbatas.** Ini adalah penyimpanan untuk data tidak terstruktur dalam jumlah masif (gambar, video, backup, log). Sangat tahan lama dan murah. Anda mengaksesnya melalui API, bukan sebagai drive lokal. Contohnya adalah Amazon S3 atau Google Cloud Storage.

Apa Selanjutnya?

Kita telah membangun infrastruktur inti: komputasi, jaringan, dan penyimpanan. Namun, bagaimana kita mengontrol siapa yang boleh melakukan apa dan bagaimana kita mengawasi semuanya?
